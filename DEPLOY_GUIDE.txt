============================================================
  AWS EC2 배포 가이드 — AIMZ EDU (Streamlit)
  ※ 모든 명령은 Windows PowerShell에서 실행
============================================================

[ Step 0: AWS 준비 ]

1. AWS 계정 생성 (없다면)
   - https://aws.amazon.com 접속 → "AWS 계정 생성"
   - 이메일 + 법인카드 등록 (KRW 결제 가능)
   - 프리 티어 12개월 제공

2. AWS Console 로그인
   - https://console.aws.amazon.com
   - 리전을 "아시아 태평양 (도쿄) ap-northeast-1"로 설정
     (우측 상단 리전 선택 드롭다운)


[ Step 1: AWS CLI 설치 ]

  # Windows PowerShell (관리자 권한)
  winget install Amazon.AWSCLI

  # 설치 후 터미널 재시작, 확인
  aws --version

  # 자격증명 설정
  aws configure

  aws configure 실행하면:
  - AWS Access Key ID     → IAM에서 발급 (아래 참고)
  - AWS Secret Access Key → IAM에서 발급
  - Default region name   → ap-northeast-1
  - Default output format → json

  ※ Access Key 발급 (상세):

    1) AWS Console 우측 상단 → 계정 이름 클릭 → "보안 자격 증명"
       (또는 https://console.aws.amazon.com/iam/home#/security_credentials)

    2) "액세스 키" 섹션까지 스크롤 → "액세스 키 만들기" 클릭

    3) 사용 사례 선택 화면:
       → "Command Line Interface(CLI)" 선택
       → 하단 체크박스 "위의 권장 사항을 이해하며..." 체크
       → "다음" 클릭

    4) 설명 태그 (선택): "aimz-edu-deploy" 등 메모 입력 → "액세스 키 만들기"

    5) 키가 생성되면 두 값이 표시됨:
       - Access Key ID:     AKIA... 로 시작하는 문자열
       - Secret Access Key: 긴 영문+숫자 문자열

       ⚠️ Secret Access Key는 이 화면에서만 확인 가능!
       반드시 ".csv 파일 다운로드" 클릭하여 저장하거나
       두 값을 안전한 곳에 복사해 둘 것.

    6) aws configure에 위 두 값을 붙여넣기하면 완료.

    ※ 루트 계정 키 대신 IAM 사용자 키 사용을 권장하나,
      개인/소규모 프로젝트에서는 루트 계정 키로도 무방함.


[ Step 2: EC2 인스턴스 생성 ]

  1) EC2 콘솔 접속
     https://ap-northeast-1.console.aws.amazon.com/ec2

  2) "인스턴스 시작" 클릭

  3) 설정:
     - 이름: aimz-edu
     - AMI: Amazon Linux 2023 (기본 선택됨)
     - 인스턴스 유형: t3.small (2 vCPU, 2GB) — 프리티어는 t2.micro
       ※ t2.micro(1 vCPU, 1GB)로도 가능하나 빌드 시 느림
     - 키 페어: "새 키 페어 생성" 클릭
       → 이름: aimz-edu-key
       → 유형: RSA
       → 형식: .pem
       → "키 페어 생성" → aimz-edu-key.pem 파일 자동 다운로드
       ⚠️ 이 파일은 다시 다운로드할 수 없음! 안전한 곳에 보관

  4) 네트워크 설정 → "편집" 클릭:
     - "보안 그룹 생성" 선택
     - 보안 그룹 이름: aimz-edu-sg
     - 규칙 추가 (아래 3개):

       규칙 1 (SSH):    유형=SSH,    포트=22,   소스=내 IP
       규칙 2 (HTTP):   유형=HTTP,   포트=80,   소스=0.0.0.0/0
       규칙 3 (HTTPS):  유형=HTTPS,  포트=443,  소스=0.0.0.0/0
       규칙 4 (Streamlit): 유형=사용자 지정 TCP, 포트=8501, 소스=0.0.0.0/0

     ※ 포트 8501은 Streamlit 기본 포트. 나중에 80→8501 리다이렉트 설정 가능

  5) 스토리지: 20 GiB gp3 (기본 8GB → 20GB로 변경, Docker 이미지 용)

  6) "인스턴스 시작" 클릭 → 1~2분 후 실행됨


[ Step 3: EC2 접속 + Docker 설치 ]

  ※ {퍼블릭IP}는 EC2 콘솔 → 인스턴스 → "퍼블릭 IPv4 주소"에서 확인

  # 1) SSH 접속 (PowerShell에서)
  ssh -i "C:\Users\{사용자}\Downloads\aimz-edu-key.pem" ec2-user@{퍼블릭IP}

  ※ "Are you sure you want to continue connecting?" → yes 입력
  ※ 권한 오류 시: 탐색기에서 .pem 파일 → 속성 → 보안 → 고급
     → 상속 사용 안 함 → 현재 사용자만 읽기 권한

  # 2) Docker 설치 (EC2 내부에서 실행)
  sudo yum update -y
  sudo yum install -y docker
  sudo systemctl start docker
  sudo systemctl enable docker
  sudo usermod -aG docker ec2-user

  # 3) 그룹 적용을 위해 재접속
  exit
  ssh -i "C:\Users\{사용자}\Downloads\aimz-edu-key.pem" ec2-user@{퍼블릭IP}

  # 4) Docker 확인
  docker --version


[ Step 4: 로컬에서 이미지 빌드 + EC2로 전송 ]

  === 방법 A: 이미지 파일 직접 전송 (ECR 없이, 간단) ===

  # 로컬 PowerShell에서:

  # 1) 이미지 빌드
  cd d:/StreamLit/TestBase
  docker build -t aimz-edu .

  # 2) 이미지를 tar 파일로 저장
  docker save aimz-edu:latest -o aimz-edu.tar

  # 3) EC2로 전송 (SCP)
  scp -i "C:\Users\{사용자}\Downloads\aimz-edu-key.pem" aimz-edu.tar ec2-user@{퍼블릭IP}:~/

  # 4) EC2에서 이미지 로드 (SSH 접속 후)
  docker load -i ~/aimz-edu.tar


  === 방법 B: ECR 사용 (이미 ECR에 푸시해뒀다면) ===

  # EC2 내부에서:
  # 1) AWS CLI 설치
  sudo yum install -y aws-cli

  # 2) ECR 로그인
  aws ecr get-login-password --region ap-northeast-1 | docker login --username AWS --password-stdin {계정ID}.dkr.ecr.ap-northeast-1.amazonaws.com

  # 3) 이미지 풀
  docker pull {계정ID}.dkr.ecr.ap-northeast-1.amazonaws.com/aimz-edu:latest
  docker tag {계정ID}.dkr.ecr.ap-northeast-1.amazonaws.com/aimz-edu:latest aimz-edu:latest


[ Step 5: 환경변수 파일 생성 + 컨테이너 실행 ]

  # EC2 내부 (SSH 접속 상태)에서:

  # 1) 환경변수 파일 생성
  cat > ~/env.list << 'ENVEOF'
KEY_POOL_JSON={"openai":[{"name":"openai-1","api_key":"sk-...","concurrency_limit":10,"rpm_limit":500,"priority":10,"tenant_scope":"*","is_active":true}],"kling":[...],"elevenlabs":[...],"google_imagen":[...],"google_veo":[...],"grok":[...]}
SUNO_ACCOUNTS_JSON=[{"id":0,"email":"","password":"","memo":"배정 없음"}]
OPENAI_MODEL=gpt-4o-mini
KLING_MODEL=kling-v2.6-std
ELEVENLABS_MODEL=eleven_multilingual_v2
GOOGLE_IMAGEN_MODEL=gemini-2.5-flash-image
GOOGLE_VEO_MODEL=veo-3.1-generate-preview
GROK_MODEL=grok-imagine-video
ENVEOF

  ※ KEY_POOL_JSON 값은 secrets.toml의 KEY_POOL_JSON 내용을
    한 줄로 이어붙여서 넣어야 함 (줄바꿈 없이)

  # 2) DB 데이터 디렉토리 생성 + 컨테이너 실행 (볼륨 마운트)
  mkdir -p ~/data
  docker run -d --name aimz-edu --restart unless-stopped --env-file ~/env.list -v ~/data:/app/data -p 8501:8080 aimz-edu:latest

  ※ -v ~/data:/app/data: DB 파일을 호스트에 저장하여 컨테이너 교체 시에도 유지
  ※ 컨테이너는 appuser(uid=1000)로 실행되므로 ~/data 소유자가 ec2-user(uid=1000)이면 권한 호환

  # 3) 로그 확인
  docker logs -f aimz-edu

  "You can now view your Streamlit app" 메시지가 나오면 성공!
  Ctrl+C로 로그 빠져나옴 (컨테이너는 계속 실행됨)


[ Step 6: 접속 확인 ]

  브라우저에서:
  http://{퍼블릭IP}:8501

  로그인 화면이 나오면 성공!


[ 이후 업데이트할 때 ]

  코드 수정 후:

  === 로컬 PowerShell ===

  # 1) 이미지 다시 빌드
  cd d:/StreamLit/TestBase
  docker build -t aimz-edu .
  docker save aimz-edu:latest -o aimz-edu.tar
  scp -i "C:\Users\{사용자}\Downloads\aimz-edu-key.pem" aimz-edu.tar ec2-user@{퍼블릭IP}:~/

  === EC2 (SSH 접속 후) ===

  # 2) 기존 컨테이너 정지 + 볼륨 권한 복구 + 새 이미지 로드 + 재시작
  docker stop aimz-edu && docker rm aimz-edu
  sudo chown -R 1000:1000 ~/data    # docker cp가 root 소유로 생성한 파일 권한 복구
  docker load -i ~/aimz-edu.tar
  docker run -d --name aimz-edu --restart unless-stopped --env-file ~/env.list -v ~/data:/app/data -p 8501:8080 aimz-edu:latest

  환경변수(env.list)는 EC2에 남아있으므로 다시 만들 필요 없음.
  ※ sudo chown이 필요한 이유: docker cp로 DB 백업 시 root 소유로 파일이 생성되어
    컨테이너의 appuser(uid=1000)가 쓸 수 없게 됨. 배포 스크립트(z_deploy.bat 등)는
    이 명령을 자동으로 포함하고 있음.

[ 커스텀 도메인 + HTTPS 설정 ]

  ※ 이 과정을 완료하면 브라우저의 "안전하지 않음" 경고가 사라지고
    자물쇠(🔒) 아이콘이 표시됨
  ※ 도메인이 없으면 http://{퍼블릭IP}:8501 으로 계속 사용 가능하나
    브라우저에 "안전하지 않음" 경고가 항상 표시됨

  ── 1단계: 탄력적 IP(Elastic IP) 할당 ──

  EC2 퍼블릭 IP는 인스턴스를 중지/시작하면 바뀜.
  도메인을 연결하려면 IP를 고정해야 함.

  1) EC2 콘솔 → 좌측 "네트워크 및 보안" → "탄력적 IP"
  2) "탄력적 IP 주소 할당" 클릭
  3) "할당" 클릭 (기본 설정 그대로)
  4) 생성된 IP 선택 → "작업" → "탄력적 IP 주소 연결"
  5) 인스턴스: aimz-edu 선택 → "연결"

  ※ 이제 이 IP가 고정됨. SSH 접속도 이 IP로 변경:
     ssh -i "C:\Users\{사용자}\Downloads\aimz-edu-key.pem" ec2-user@{탄력적IP}
  ※ 탄력적 IP는 EC2에 연결되어 있으면 무료, 미연결 시 $0.005/시간

  ── 2단계: 도메인 준비 ──

  도메인이 없다면 먼저 구매:
  - 가비아 (https://www.gabia.com) — 한국 도메인 등록업체
  - Cloudflare (https://www.cloudflare.com) — .com 도메인 저렴
  - AWS Route53 — AWS 내에서 직접 관리 가능

  ※ .com 도메인 기준 연간 약 $10~15 (약 13,000~20,000원)

  ── 3단계: 도메인 DNS 설정 ──

  도메인 관리 사이트의 DNS 설정 페이지에서:

  1) DNS 관리 → A 레코드 추가
     - 호스트/이름: @ (루트 도메인) 또는 edu (서브도메인)
     - 유형: A
     - 값/대상: {탄력적IP}  (1단계에서 할당한 고정 IP)
     - TTL: 300 (5분)

  예시:
     edu.example.com  →  A  →  13.113.xxx.xxx
     example.com      →  A  →  13.113.xxx.xxx

  ※ DNS 반영까지 5분~24시간 소요 (보통 5~30분)

  확인 방법 (로컬 PowerShell에서):
     nslookup your-domain.com
     → 탄력적 IP가 나오면 DNS 설정 완료

  ── 4단계: EC2 보안 그룹 확인 ──

  Step 2에서 이미 80, 443 포트를 열었으면 이 단계는 건너뛰기.

  EC2 콘솔 → 인스턴스 → 보안 → 보안 그룹 클릭 → "인바운드 규칙 편집"

  아래 규칙이 있는지 확인 (없으면 추가):
     HTTP   | TCP | 80  | 0.0.0.0/0
     HTTPS  | TCP | 443 | 0.0.0.0/0

  ── 5단계: Nginx 설치 + 리버스 프록시 설정 (EC2 SSH) ──

  Nginx가 HTTPS를 처리하고 → Streamlit(8501)로 요청을 전달하는 구조.

  [인터넷] → :443(HTTPS) → [Nginx] → :8501 → [Streamlit]

  # EC2에 SSH 접속 후:

  # 1) Nginx 설치
  sudo yum install -y nginx

  # 2) Nginx 설정 파일 생성
  #    ⚠️ your-domain.com 을 실제 도메인으로 변경할 것!

  sudo tee /etc/nginx/conf.d/aimz-edu.conf > /dev/null << 'NGEOF'
  server {
      listen 80;
      server_name your-domain.com;

      location / {
          proxy_pass http://127.0.0.1:8501;
          proxy_http_version 1.1;
          proxy_set_header Upgrade $http_upgrade;
          proxy_set_header Connection "upgrade";
          proxy_set_header Host $host;
          proxy_set_header X-Real-IP $remote_addr;
          proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
          proxy_set_header X-Forwarded-Proto $scheme;
          proxy_read_timeout 86400;
      }

      # Streamlit WebSocket 경로
      location /_stcore/stream {
          proxy_pass http://127.0.0.1:8501/_stcore/stream;
          proxy_http_version 1.1;
          proxy_set_header Upgrade $http_upgrade;
          proxy_set_header Connection "upgrade";
          proxy_set_header Host $host;
          proxy_read_timeout 86400;
      }
  }
  NGEOF

  # 3) 설정 문법 확인
  sudo nginx -t

  # "syntax is ok" 가 나와야 함. 에러가 나면 설정 파일 확인.

  # 4) Nginx 시작 + 부팅 시 자동 시작
  sudo systemctl start nginx
  sudo systemctl enable nginx

  # 5) HTTP 접속 테스트
  #    브라우저에서 http://your-domain.com 으로 접속해서
  #    Streamlit 화면이 나오는지 확인. (아직 HTTPS는 아님)

  ── 6단계: Let's Encrypt SSL 인증서 발급 (무료) ──

  # 1) certbot 설치
  sudo yum install -y certbot python3-certbot-nginx

  #    ※ Amazon Linux 2023에서 certbot이 안 되면:
  #       sudo yum install -y python3-pip
  #       sudo pip3 install certbot certbot-nginx

  # 2) SSL 인증서 발급
  #    ⚠️ your-domain.com 을 실제 도메인으로 변경!
  #    ⚠️ 5단계에서 http 접속이 되는 상태여야 함!

  sudo certbot --nginx -d your-domain.com

  #    실행하면 아래 질문이 나옴:
  #
  #    "Enter email address" → 관리자 이메일 입력 (인증서 만료 알림용)
  #    "Do you agree to the Terms of Service?" → Y
  #    "Would you be willing to share your email?" → N (선택)
  #
  #    성공하면:
  #    "Congratulations! Your certificate and chain have been saved at:
  #     /etc/letsencrypt/live/your-domain.com/fullchain.pem"
  #
  #    certbot이 자동으로 Nginx 설정을 HTTPS용으로 수정해줌.

  # 3) 인증서 자동 갱신 설정
  #    Let's Encrypt 인증서는 90일마다 갱신 필요.
  #    certbot이 자동 갱신 타이머를 설치하지만, 확인:

  sudo certbot renew --dry-run

  #    "Congratulations, all simulated renewals succeeded" 가 나오면 성공.

  #    ※ 자동 갱신이 안 되어있다면 수동으로 cron 등록:
  #       sudo crontab -e
  #       아래 한 줄 추가 (매일 새벽 3시에 갱신 시도):
  #       0 3 * * * certbot renew --quiet && systemctl reload nginx

  ── 7단계: 접속 확인 ──

  브라우저에서 https://your-domain.com 으로 접속.

  ✅ 확인 사항:
  - 주소창에 자물쇠(🔒) 아이콘이 보이면 HTTPS 설정 완료
  - "안전하지 않음" 경고가 사라져야 함
  - http://your-domain.com 으로 접속해도 자동으로 https로 리다이렉트됨

  ※ 기존 8501 포트 직접 접속도 계속 가능:
     http://{탄력적IP}:8501
     (단, 이 주소는 HTTPS가 아니므로 "안전하지 않음" 경고 표시됨)

  ── 문제 해결 ──

  Q: certbot 실행 시 "Could not automatically find a matching server block" 에러
  A: Nginx 설정 파일의 server_name이 실제 도메인과 정확히 일치하는지 확인.
     sudo nano /etc/nginx/conf.d/aimz-edu.conf 로 수정 후 sudo nginx -t

  Q: certbot 실행 시 "Connection refused" 또는 "Timeout" 에러
  A: 보안 그룹에 80번 포트가 열려있는지 확인.
     DNS가 올바른 IP를 가리키고 있는지 nslookup으로 확인.

  Q: https 접속 시 "ERR_CONNECTION_REFUSED"
  A: 보안 그룹에 443번 포트가 열려있는지 확인.
     sudo systemctl status nginx 로 Nginx가 실행 중인지 확인.

  Q: 인증서 갱신 실패 알림 이메일을 받았을 때
  A: sudo certbot renew --force-renewal 실행 후
     sudo systemctl reload nginx

  EC2 요금 (ap-northeast-1 기준):
  - t2.micro: 프리 티어 (12개월간 월 750시간 무료)
  - t3.small: ~$0.0272/시간 → 약 $20/월 (상시 운영)
  - t3.medium: ~$0.0544/시간 → 약 $40/월

  추가 비용:
  - EBS 스토리지: 20GB gp3 = ~$1.6/월
  - 데이터 전송: 월 100GB 무료, 이후 $0.114/GB
  - 탄력적 IP: EC2에 연결된 상태면 무료, 미연결 시 $0.005/시간

  ※ 사용하지 않을 때는 EC2 콘솔에서 "인스턴스 중지"하면 과금 멈춤
    (EBS 스토리지 비용만 발생)
