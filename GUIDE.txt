========================================
  AI API 교육 플랫폼 — 운영 가이드
========================================

이 문서는 개발 지식 없이도 플랫폼을 도입·운영할 수 있도록 작성되었습니다.
대상: 학교 관리자(교수/교직원), 수업 운영자(조교/TA)


[ 1. 이 플랫폼은 무엇인가요? ]

학생들이 다양한 AI API를 직접 체험할 수 있는 웹 교육 도구입니다.
별도 설치 없이 웹 브라우저에서 접속하며, 다음 AI 서비스를 제공합니다:

  - GPT 대화       : ChatGPT와 대화 (OpenAI)
  - 이미지 생성     : Google Gemini — MJ탭, NanoBanana탭
  - 비디오 생성     : Kling AI / Google Veo — Kling탭 (2개)
  - 음성 합성(TTS)  : ElevenLabs
  - 음악 생성       : Suno

학생은 ID/PW로 로그인한 뒤 사이드바에서 원하는 탭을 선택하여 AI를 사용합니다.
관리자는 별도 관리 화면에서 사용자 관리, 사용 현황 모니터링 등을 할 수 있습니다.


[ 2. 사용자 역할 ]

  관리자(admin)    : 전체 시스템 관리. 사용자 생성/삭제, 모니터링, DB 관리 등
  교수(teacher)    : 수업 관련 기능 사용 + 플로팅 채팅
  학생(student)    : AI 탭 체험 + 플로팅 채팅
  열람자(viewer)   : 모니터링, 실행 이력, 부하테스트 결과 열람 (읽기 전용)

최초 접속 시 관리자 계정을 하나 만들게 됩니다.
이후 관리자가 학생/교수 계정을 직접 생성합니다.


[ 3. 최초 설정 (관리자용) ]

3-1. 플랫폼 접속
  - 배포된 URL(Cloud Run 또는 Streamlit Cloud)로 접속합니다.
  - 최초 접속 시 "초기 관리자 생성" 화면이 나타납니다.
  - 관리자 ID, PW, 학교 ID를 입력하고 [생성] 버튼을 누릅니다.

3-2. 학생 계정 생성
  - 관리자로 로그인 → 사이드바에서 "계정 관리" 선택
  - 계정 추가 영역에서 ID, PW, 역할(student), 학교를 지정하고 추가합니다.

3-3. Suno 계정 배정 (음악 생성 사용 시)
  - "계정 관리"에서 학생 편집 → Suno 계정 선택
  - Suno는 개인 계정 기반이므로 학생에게 1:1 배정이 필요합니다.


[ 4. 로그인 화면 ]

접속하면 로그인 화면이 나타납니다.
  - ID 입력란
  - PW 입력란
  - [로그인] 버튼

로그인에 성공하면 사이드바에 프로필 카드가 표시되고,
역할에 따라 다른 화면이 나타납니다.
  - 관리자(admin): 관리 메뉴 6개
  - 열람자(viewer): 모니터링 메뉴 3개
  - 교수/학생(teacher/student): AI 체험 탭 목록


[ 5. 사이드바 ]

로그인 후 왼쪽 사이드바에 다음 요소가 표시됩니다:

  프로필 카드
    - 로고, 사용자 ID, 역할 배지, 소속 학교명
    - [로그아웃] 버튼

  페이지 선택 (라디오 버튼)
    - 관리자: 모니터링, 키풀 상태, 실행 기록, 부하테스트, 계정 관리, DB 관리
    - 열람자: 모니터링, 실행 기록, 부하테스트 결과
    - 교수/학생: GPT, Midjourney, NanoBanana, Kling, Kling(Veo), ElevenLabs, Suno
      (학교 설정에 따라 보이는 탭이 다를 수 있음)

  하단 고정
    - "MOCK 모드" 토글 (교수/학생 전용)
      켜면 실제 API를 호출하지 않고 UI 동작만 테스트합니다.


[ 6. AI 체험 탭 — 상세 기능 안내 ]

사이드바에서 탭을 선택하면 메인 화면에 해당 AI 서비스가 표시됩니다.

────────────────────────────────────
  6-1. GPT (ChatGPT 대화)
────────────────────────────────────

  왼쪽 대화 목록:
    - [+ New Chat] 버튼: 새 대화 시작
    - 이전 대화 목록: 클릭하면 해당 대화로 전환
    - 각 대화 항목에 마우스를 올리면 삭제 버튼 표시

  상단:
    - 모델 선택 드롭다운
      (GPT-4o mini, GPT-4o, GPT-4.1 mini, GPT-4.1, o4-mini)

  대화 영역:
    - 내 메시지(오른쪽 말풍선)와 AI 응답(왼쪽)이 번갈아 표시
    - AI 응답 중에는 점 세 개가 깜빡이는 로딩 애니메이션 표시
    - 코드 블록, 볼드, 리스트 등 마크다운 서식 지원

  하단 입력:
    - 텍스트 입력란 (Enter로 전송 가능)
    - [전송] 버튼

────────────────────────────────────
  6-2. Midjourney (이미지 생성)
────────────────────────────────────

  상단 입력 바:
    - 프롬프트 입력란: 생성할 이미지를 설명하는 텍스트 입력
    - [전송] 버튼: 이미지 생성 요청
    - [설정] 버튼(톱니바퀴 아이콘): 설정 패널 열기/닫기
    - [첨부] 버튼: 갤러리에서 기존 이미지 선택하여 첨부

  설정 패널 (설정 버튼 클릭 시 표시):
    - 모델 선택: 드롭다운에서 모델 선택 (아이콘, 설명 포함)
    - 프레임 업로드: 드래그&드롭으로 이미지 업로드
    - 사운드 이펙트: 토글 스위치로 활성화/비활성화

  갤러리:
    - 생성된 이미지가 갤러리에 표시
    - [Gallery] 버튼으로 학교 공유 갤러리 열기/닫기

────────────────────────────────────
  6-3. Kling (비디오 생성)
────────────────────────────────────

  Kling 탭은 2개가 있을 수 있습니다:
    - Kling: Kling AI API로 비디오 생성
    - Kling(Veo): Google Veo API로 비디오 생성
  두 탭의 UI는 동일하며, 사용하는 AI 엔진만 다릅니다.

  왼쪽 입력 패널:
    - 모델 선택: 드롭다운에서 모델 선택 (아이콘, 설명 포함)
    - 프레임 업로드: 시작/끝 프레임 이미지 드래그&드롭
    - 사운드: 토글 스위치로 사운드 활성화 (BETA 뱃지)
    - 프롬프트 입력: 생성할 비디오를 설명하는 텍스트
    - 하단 설정 바: 비율, 해상도, 길이 등 칩 형태로 조절

  메인 영역:
    - 생성된 비디오 표시
    - 로딩 중에는 진행 애니메이션 표시

  갤러리:
    - [Gallery] 버튼으로 학교 공유 갤러리 열기

────────────────────────────────────
  6-4. ElevenLabs (음성 합성)
────────────────────────────────────

  왼쪽 메인 패널:
    - 텍스트 입력란: 음성으로 변환할 텍스트 입력
    - 추천 문장 칩: 클릭하면 해당 문장이 입력란에 자동 입력
    - [Generate] 버튼: 음성 생성

  오른쪽 설정 패널:
    - Voice / Model 전환 탭
    - 보이스 선택: 드롭다운 (아바타 아이콘 포함)
    - 모델 선택: 드롭다운 (뱃지 표시)
    - 슬라이더:
      · Stability (안정성)
      · Similarity (유사도)
      · 기타 음성 파라미터
    - 토글 스위치: Speaker Boost 등 옵션

  생성 결과:
    - 오디오 플레이어로 생성된 음성을 바로 재생

────────────────────────────────────
  6-5. NanoBanana (이미지 생성/편집)
────────────────────────────────────

  왼쪽 세션 목록:
    - 이전 세션 목록 (클릭하여 전환)
    - 크레딧 표시 영역

  대화 영역:
    - 첫 화면: 환영 메시지 + 추천 프롬프트 칩 (클릭 가능)
    - 대화 형식으로 이미지 생성:
      · 내 메시지 (오른쪽 말풍선)
      · AI 생성 이미지 (1~4개 그리드 배열)
    - 이미지에 마우스를 올리면 액션 버튼 표시
    - 이미지 클릭: 라이트박스로 원본 크기 보기
    - 이미지별 편집 기능: 기존 이미지를 기반으로 수정 요청 가능

  하단 입력:
    - 설정 바: 톱니바퀴 버튼으로 열기/닫기
      · 비율, 모델 등 칩 형태로 선택
    - 텍스트 입력란
    - [전송] 버튼

────────────────────────────────────
  6-6. Suno (음악 생성)
────────────────────────────────────

  이 탭은 외부 Suno 서비스로 연결합니다.

  화면 표시:
    - 배정된 Suno 계정 정보 카드
      · 계정 번호, 메모
      · 이메일 주소
      · 비밀번호
    - [Suno 열기] 링크 버튼: suno.com으로 이동

  주의: Suno 계정이 배정되지 않은 학생에게는
  "배정된 Suno 계정이 없습니다" 안내 메시지가 표시됩니다.

────────────────────────────────────
  6-7. 플로팅 채팅
────────────────────────────────────

  교수(teacher)와 학생(student)에게만 표시되는 채팅 기능입니다.
  화면 오른쪽 하단에 채팅 버튼이 떠 있습니다.

    - 같은 학교 소속 사용자끼리 실시간 메시지 주고받기
    - 3초 간격 자동 새로고침
    - 수업 중 질문/공지 용도로 활용

────────────────────────────────────
  6-8. 플로팅 알림/점검 배너
────────────────────────────────────

  화면 최상단에 고정되는 알림 배너입니다. 모든 역할에게 표시됩니다.

    알림 배너 (파란색):
    - 관리자가 등록한 공지가 화면 최상단에 표시
    - X 버튼으로 닫기 가능 (브라우저 세션 동안 유지)
    - 5초마다 서버에서 새 알림 자동 확인

    점검 카운트다운 배너 (빨간색):
    - 서버 점검 예약 시 남은 시간이 실시간 표시
    - "3시간 24분 15초 후 서버 점검이 시작됩니다" 형태
    - 매초 자동 업데이트 (JS 타이머)
    - 점검 시작 시 비관리자 사용자는 자동 차단


[ 7. 수업 중 운영 (조교/TA용) ]

7-1. 학생 로그인 안내
  - 학생에게 접속 URL과 발급한 ID/PW를 전달합니다.
  - 로그인 후 사이드바에서 원하는 AI 탭을 선택합니다.

7-2. 실시간 모니터링
  - 관리자 로그인 → 사이드바에서 "모니터링" 선택
  - 실시간 지표 3개:
    · Active Jobs: 현재 실행 중인 API 작업 수
    · Active Users: 현재 활동 중인 사용자 수
    · Providers: 활성 상태인 API 제공자 수
  - 하단에 실행 중인 작업 목록 테이블 (1초 간격 자동 갱신)
  - 문제가 생긴 작업이 있으면 여기서 확인 가능합니다.

7-3. 키 풀 상태 확인
  - 사이드바에서 "키풀 상태" 선택
  - 대기열 테이블: API 키를 기다리는 요청 목록
  - 활성 리스 테이블: 현재 API 키가 할당된 작업 목록
  - 대기열이 길어지면 동시 사용자가 많다는 의미입니다.

7-4. 문제 해결
  - 학생이 "로딩이 안 끝나요": 모니터링에서 해당 작업 확인
  - 학생이 "오류가 나요": 실행 이력에서 에러 내용 확인
  - 전체적으로 느려짐: API 키 동시 사용 한도에 도달했을 수 있음


[ 8. 관리자 화면 — 상세 기능 안내 ]

관리자 로그인 시 사이드바에 7개 메뉴가 표시됩니다.
라디오 버튼으로 원하는 메뉴를 선택합니다.

────────────────────────────────────
  8-1. 모니터링
────────────────────────────────────

  실시간 지표 (1초 자동 갱신):
    - Active Jobs: 현재 진행 중인 API 작업 수
    - Active Users: 현재 활동 중인 사용자 수
    - Providers: 활성 API 제공자 수

  활성 작업 테이블:
    - 현재 실행 중인 모든 API 요청을 표로 표시
    - 사용자, 요청 유형, 시작 시각 등 확인 가능

────────────────────────────────────
  8-2. 키풀 상태
────────────────────────────────────

  대기열(Waiters) 테이블:
    - API 키 할당을 기다리는 요청 목록

  활성 리스(Leases) 테이블:
    - 현재 API 키가 할당되어 사용 중인 작업 목록

────────────────────────────────────
  8-3. 실행 기록
────────────────────────────────────

  상단 필터:
    - 사용자 필터: 전체 또는 특정 사용자 선택
    - 표시 건수: 슬라이더로 50~500건 조절

  서비스별 기록 (접기/펼치기 가능):
    - GPT 대화 기록
    - Midjourney 이미지 기록
    - Kling 비디오 기록
    - ElevenLabs 음성 기록
    - NanoBanana 이미지 기록

  각 기록 테이블:
    - "보기" 체크박스: 체크하면 상세 내용이 팝업으로 표시
    - 팝업에서 프롬프트, 생성 결과(이미지/오디오/비디오), 설정값 등 확인

────────────────────────────────────
  8-4. 부하테스트
────────────────────────────────────

  수업 전에 "학생 N명이 동시에 사용해도 되는지" 미리 확인합니다.

  설정:
    - 테스트할 AI 서비스 선택 (GPT, Midjourney 등)
    - 동시 사용자 수 설정 (예: 5, 10, 20, 30)
    - Mock/Real 모드 선택

  실행:
    - [플랜 실행] 버튼으로 테스트 시작
    - 실시간 진행률 표시

  결과:
    - 서비스별 성공률, 평균 응답 시간, P95/P99, 등급(S~F)
    - 사용자 수별 추이 차트

────────────────────────────────────
  8-5. 계정 관리
────────────────────────────────────

  계정 목록:
    - 전체 사용자를 테이블로 표시

  계정 추가:
    - 사용자 ID 입력
    - 비밀번호 입력
    - 역할 선택 (student, teacher, viewer, admin)
    - 학교 ID 선택
    - [추가] 버튼

  계정 편집:
    - 대상 계정 선택 (드롭다운)
    - 역할 변경, 학교 변경, 활성화/비활성화 토글
    - 비밀번호 재설정
    - Suno 계정 배정 (드롭다운)
    - [저장] 버튼

  계정 삭제:
    - 확인 텍스트 입력 후 [삭제] 버튼

────────────────────────────────────
  8-6. 알림/점검 관리
────────────────────────────────────

  알림 발송:
    - 메시지 입력 후 [알림 보내기] 버튼
    - 활성 알림은 항상 1개 (새 알림 등록 시 기존 자동 비활성화)
    - [알림 끄기] 버튼으로 현재 알림 비활성화

  서버 점검 예약:
    - 점검 시작 날짜/시간 입력 (KST 기준)
    - 예상 소요 시간 입력
    - 안내 메시지 입력
    - [점검 예약] 버튼으로 등록
    - 점검 시간 1시간 전부터 사용자에게 카운트다운 배너 자동 표시
    - 점검 시작 시 비관리자 사용자 자동 차단
    - [서비스 재개] 버튼으로 점검 해제

────────────────────────────────────
  8-7. DB 관리
────────────────────────────────────

  테이블 현황:
    - 각 테이블의 이름, 레코드 수, 자동 삭제 상태를 표로 표시

  수동 삭제:
    - 테이블 선택 (드롭다운)
    - "N일 이전" 입력
    - 해당 기간의 오래된 기록 수 표시
    - 확인 텍스트 입력 후 [삭제 실행] 버튼

  자동 삭제 설정:
    - 테이블별 자동 삭제 일수 입력 (숫자 입력란)
    - 0 = 자동 삭제 비활성 (기본값)
    - 예: 90 입력 → 90일 이전 기록 자동 삭제
    - [설정 저장] 버튼
    - [지금 자동 삭제 실행] 버튼


[ 9. 열람자(viewer) 화면 ]

열람자 로그인 시 사이드바에 3개 메뉴가 표시됩니다.
관리자와 동일한 형태이지만 읽기 전용입니다.

  (1) 모니터링: 실시간 활성 작업 확인 (수정 불가)
  (2) 실행 기록: 사용자별 API 사용 기록 열람 + 상세 보기
  (3) 부하테스트 결과: 과거 부하 테스트 결과 조회


[ 10. 파일 편집이 필요한 경우 ]

아래 항목은 관리 화면에서 설정할 수 없고, 직접 파일을 수정해야 합니다.
개발 담당자 또는 배포 환경(Cloud Run 환경변수 / Streamlit Cloud Secrets)에서 수정합니다.

  (1) 새 학교 추가
      - tenants 폴더에 JSON 파일 생성
      - 학교명, 사용할 탭 목록, 로고 경로 등 지정

  (2) API 키 추가/변경
      - Cloud Run: 환경변수에서 KEY_POOL_JSON 수정
        Streamlit Cloud: Settings → Secrets 에서 KEY_POOL_JSON 수정
      - 각 AI 서비스별 키, 동시 사용 한도, 분당 요청 한도 설정

  (3) Vertex AI 설정 (이미지/비디오 생성용)
      - Google Cloud Console에서 서비스 계정 생성 및 JSON 키 발급 필요
      - 설정 방법:
        a) GCP Console → IAM 및 관리자 → 서비스 계정 → 계정 만들기
        b) 역할: Vertex AI 사용자(Vertex AI User) 부여
        c) 키 → 키 추가 → JSON → 다운로드
        d) Secrets에 아래 항목 설정:
           VERTEX_SA_JSON   = 다운로드한 JSON 파일 내용 전체
           VERTEX_PROJECT_ID = GCP 프로젝트 ID
           VERTEX_LOCATION   = 리전 (기본값: us-central1)
      - KEY_POOL_JSON 내 google_imagen, google_veo 항목은
        "sa_json": "__VERTEX__" 로 설정하면 위 값을 자동 참조합니다.

  (4) Suno 계정 추가
      - Cloud Run: 환경변수에서 SUNO_ACCOUNTS_JSON 수정
        Streamlit Cloud: Settings → Secrets 에서 SUNO_ACCOUNTS_JSON 수정

  (5) 탭 노출 제어
      - tenants JSON의 enabled_features에서 탭 활성화/비활성화
      - 예: "tab.kling"만 넣으면 Kling API 탭만 표시
            "tab.kling_veo"만 넣으면 Veo 탭만 표시
            둘 다 넣으면 Kling 탭 2개 표시


[ 11. 용어 설명 ]

  API 키       : AI 서비스를 사용하기 위한 인증 코드.
                 유료이며, 키 하나당 동시 사용 가능 인원이 제한됩니다.

  Vertex AI    : Google Cloud의 AI 플랫폼. 서비스 계정(SA) 인증 방식으로
                 API Key 방식보다 일일 요청 한도(RPD)가 훨씬 넓습니다.
                 이미지 생성(Gemini)과 비디오 생성(Veo)에 사용됩니다.

  서비스 계정   : Google Cloud에서 API를 호출하기 위한 인증 주체.
  (Service      JSON 키 파일을 발급받아 secrets에 등록합니다.
   Account)     토큰은 1시간마다 자동 갱신됩니다.

  동시 사용 한도 : 하나의 키로 동시에 요청할 수 있는 최대 인원.
  (concurrency)  예: 한도가 5이면 5명까지 동시에 사용 가능.
                 초과 시 대기열에서 순서를 기다립니다.

  RPM          : 분당 요청 한도 (Requests Per Minute).
                 1분 안에 보낼 수 있는 최대 요청 수.

  RPD          : 일일 요청 한도 (Requests Per Day).
                 하루에 보낼 수 있는 최대 요청 수. API Key 방식은 한도가
                 낮아 Vertex AI 방식으로 전환하여 해결합니다.

  테넌트        : 학교 단위 설정. 학교마다 다른 탭, 로고, 제목 지정 가능.

  Mock 모드     : 실제 API를 호출하지 않는 테스트 모드.
                 API 비용 없이 UI 동작만 확인할 때 사용.

  Turso        : 원격 데이터베이스 서비스. Streamlit Cloud 배포 시 사용.
                 로컬 DB(SQLite)와 자동 동기화됩니다.


[ 12. 자주 묻는 질문 ]

  Q: 학생 수가 늘어나면 어떻게 하나요?
  A: API 키를 추가하면 동시 수용 인원이 늘어납니다.
     예: 동시 한도 5인 키 2개 = 총 10명 동시 사용 가능.

  Q: 특정 학교에서 일부 탭만 보여주고 싶어요.
  A: 해당 학교의 tenants JSON에서 enabled_features를 수정합니다.

  Q: Kling 탭이 2개인데 차이가 뭔가요?
  A: 같은 화면이지만 사용하는 AI 엔진이 다릅니다.
     하나는 Kling AI, 다른 하나는 Google Veo로 비디오를 생성합니다.
     학교 설정에서 하나만 활성화하거나 둘 다 활성화할 수 있습니다.

  Q: 학생이 비밀번호를 잊었어요.
  A: 관리자 → 계정 관리 → 해당 학생 선택 → 비밀번호 재설정.

  Q: API 비용이 걱정돼요.
  A: Mock 모드로 먼저 테스트하세요. 실제 API 호출 없이 UI를 확인할 수 있습니다.
     또한 부하 테스트로 필요한 키 수를 사전에 파악할 수 있습니다.

  Q: 이미지/비디오 생성이 "RPD 한도 초과" 오류가 나요.
  A: API Key 방식은 일일 요청 한도가 낮습니다.
     Vertex AI 서비스 계정 방식으로 설정하면 한도가 크게 늘어납니다.
     섹션 10-(3) "Vertex AI 설정"을 참고하세요.

  Q: Vertex AI 토큰이 만료되었다고 나와요.
  A: 서비스 계정 토큰은 1시간마다 자동 갱신됩니다.
     VERTEX_SA_JSON 값이 올바른 서비스 계정 JSON인지 확인하세요.

  Q: 수업이 끝난 후 데이터를 정리하고 싶어요.
  A: 관리자 → DB 관리에서 수동 삭제하거나,
     자동 삭제 주기를 설정하면 오래된 기록이 자동으로 정리됩니다.

  Q: 플로팅 채팅은 누가 볼 수 있나요?
  A: 교수(teacher)와 학생(student)만 사용할 수 있습니다.
     같은 학교 소속 사용자끼리 메시지를 주고받습니다.


  Q: Cloud Run에 배포하려면 어떻게 하나요?
  A: DEPLOY_GUIDE.txt 파일에 상세 배포 순서가 안내되어 있습니다.
     Google Cloud CLI 설치 → gcloud run deploy 명령으로 배포합니다.
     환경변수로 API 키와 DB 정보를 설정합니다.

  Q: Cloud Run과 Streamlit Cloud 중 어떤 것을 사용해야 하나요?
  A: Cloud Run은 커스텀 도메인, 더 큰 메모리, 안정적인 운영에 유리합니다.
     Streamlit Cloud는 무료로 간편하게 배포할 수 있지만 제약이 있습니다.

  Q: 알림은 어떻게 보내나요?
  A: 관리자 → 알림/점검 관리에서 메시지를 입력하고 [알림 보내기] 버튼을 누릅니다.
     모든 사용자 화면 상단에 즉시 파란색 배너가 표시됩니다.

  Q: 서버 점검은 어떻게 예약하나요?
  A: 관리자 → 알림/점검 관리에서 날짜/시간(KST)을 지정하고 [점검 예약]을 누릅니다.
     점검 1시간 전부터 사용자에게 카운트다운이 표시되고,
     점검 시간이 되면 비관리자 사용자가 자동 차단됩니다.
     점검 완료 후 [서비스 재개] 버튼을 누르면 정상 운영됩니다.
